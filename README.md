# Lost Pet Search

Учебный сервис для поиска животных в картотеках потерянных животных по фотографии.

## Описание

Пользователь загружает фотографию животного через Streamlit-интерфейс.
Frontend отправляет изображение в FastAPI backend.
Backend сравнивает загруженное изображение с изображениями животных из SQLite-базы и возвращает наиболее похожие записи.

Проект демонстрирует минимальную архитектуру ML-сервиса с разделением на frontend, backend, базу данных и модуль обработки изображений.

## Архитектура

* `frontend/` — интерфейс пользователя на Streamlit.
* `backend/` — API на FastAPI.
* `backend/ml.py` — логика сравнения изображений.
* `backend/database.py` — работа с SQLite.
* `parser/collect_pet911_links.py` — сбор ссылок на объявления.
* `parser/load_pet911_csv.py` — загрузка данных в SQLite.
* `init_db.py` — создание тестовой базы данных.
* `data/images/` — изображения животных.
* `pets.db` — SQLite-база.

## ML-часть

В текущей версии проекта используется упрощённый подход к сравнению изображений.

Каждое изображение преобразуется в числовой вектор на основе цветовой гистограммы.
После этого рассчитывается cosine similarity между загруженным изображением и изображениями из базы.
Результаты сортируются по степени похожести и возвращаются пользователю.

Такой подход позволяет показать полный pipeline ML-сервиса без обучения тяжёлой модели:

```text
изображение → признаки → сравнение → похожие результаты
```

## Данные

В проект добавлен импорт данных из внешней картотеки Pet911.

Скрипт `parser/collect_pet911_links.py` собирает ссылки и описания объявлений.
Скрипт `parser/load_pet911_csv.py` загружает данные в SQLite-базу.

Если прямые изображения из внешнего источника недоступны, можно использовать локальные изображения из папки `data/images`.

## Установка

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## Инициализация базы

Создание тестовой базы:

```bash
python init_db.py
```

Загрузка данных из CSV:

```bash
python parser/load_pet911_csv.py
```

## Запуск backend

В первом терминале:

```bash
source venv/bin/activate
uvicorn backend.api:app --reload
```

Backend будет доступен по адресу:

```text
http://localhost:8000
```

Swagger-документация:

```text
http://localhost:8000/docs
```

## Запуск frontend

Во втором терминале:

```bash
source venv/bin/activate
streamlit run frontend/app.py
```

Frontend будет доступен по адресу:

```text
http://localhost:8501
```

## API endpoints

### GET `/health`

Проверка доступности backend.

### GET `/pets`

Получение списка животных из базы.

### POST `/search`

Поиск похожих животных по загруженной фотографии.

## Пользовательский сценарий

1. Пользователь открывает Streamlit-приложение.
2. Загружает фотографию животного.
3. Frontend отправляет изображение в FastAPI backend.
4. Backend сравнивает фото с изображениями из базы.
5. Пользователь видит похожие объявления, процент похожести, описание и ссылку на источник.

## Результат

В результате реализован минимальный сервис поиска потерянных животных по фотографии с разделением на frontend и backend.
